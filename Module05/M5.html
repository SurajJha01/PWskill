Q1 List out the features of HTML5

  1. New Semantic Elements :- < header>, < footer>, < article>, < section>, < aside>, < nav>, < figure>, < figcaption> improve readability and SEO.
  2. Multimedia Support :- Native support for audio and video using < audio> and < video> tags without requiring third-party plugins like Flash.
  3. Canvas and SVG for Graphics :-
    • < canvas> allows drawing graphics and animations via JavaScript.
    • SVG (Scalable Vector Graphics) enables rendering vector-based graphics.
  4. Form Enhancements :-
    • New input types: email, date, time, url, number, range, search, tel, color.
    • Built-in form validation without JavaScript.
    • Placeholder attribute for input fields.
  5. Local Storage and Offline Support :-
    • LocalStorage and SessionStorage APIs for storing data on the client-side.
    • Web SQL Database and IndexedDB for structured data storage.
    • Application Cache (AppCache) for offline web applications (deprecated in favor of Service Workers).

HTML entities are special codes used to represent characters that either have a reserved meaning in HTML (like <, >, and &) or are not easily typed on a keyboard (like ¬© or ¬Æ). They ensure that special characters are displayed correctly in web pages.

Q3 Define accessibility in the context of web development. Discuss why it's essential to create accessible website and how it benefits different user groups

Web accessibility refers to designing and developing websites, tools, and technologies in a way that allows people of all abilities, including those with disabilities, to use them effectively. It ensures that all users, including those with visual, auditory, motor, or cognitive impairments, can perceive, understand, navigate, and interact with web content.
  1. Inclusivity & Equal Access Everyone, including individuals with disabilities, should have equal access to digital content and services.
  2. Legal & Compliance Requirements Many countries have accessibility laws (e.g., ADA in the U.S., WCAG guidelines, EU Accessibility Act), and non-compliance can lead to legal consequences.
  3. Improved User Experience Accessible design often results in better usability for all users, including those with temporary impairments (e.g., a broken arm, poor lighting conditions).
  4. SEO & Business Benefits
    • Search engines favor well-structured, accessible websites, improving SEO rankings.
    • It increases potential reach, benefiting businesses and organizations.
  5. Future-Proofing & Compatibility Accessible sites work better across various devices, including mobile phones, smart speakers, and assistive technologies like screen readers

Q4. List any 3 ways which helps us in improving the accessibility of HTML

  1. Use Semantic HTML Elements
    • Semantic tags like <header>, <nav>, <main>, <article>, and <footer> improve readability and help assistive technologies (e.g., screen readers) understand page structure.
    • Example
      html 
      
                    <nav>
                        <ul>
                            <li><a href="#home">Home</a></li>
                            <li><a href="#about">About</a></li>
                        </ul>
                    </nav>
      Copied!
    • Benefit :-Helps users navigate the content more efficiently
  2. Add Alternative Text for Images (alt Attribute) :-
    • The alt attribute provides a text alternative for images, which is crucial for visually impaired users relying on screen readers.
    • Example
      html 
      
                                    <img src="logo.png" alt="Company logo">
      Copied!
    • Benefit:- Ensures that users who cannot see the image still understand its purpose.
  3. Ensure Keyboard Navigation & Focus Management
    • Users should be able to navigate the site without a mouse, using only the keyboard (Tab, Enter, Arrow keys).
    • Use the tabindex attribute to control focus order and ensure interactive elements (buttons, forms, links) are accessible.
    • Example
      html 
      
                            <button tabindex="0"> Button </button>
      Copied!
    • Benefit:- Helps users with motor impairments or those using assistive devices like screen readers.
